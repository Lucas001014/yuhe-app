## 三、截图拍摄方法

### 方法 1：使用模拟器截图（推荐）

#### Android 模拟器

```bash
# 1. 启动模拟器
emulator -avd <模拟器名称> &

# 2. 进入截图模式（按 Ctrl + S 或使用菜单）
# 或使用命令行截图
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png ~/Desktop/

# 3. 调整尺寸
# 使用图片编辑工具调整为 1080x1920px
```

#### iOS 模拟器

```bash
# 1. 启动模拟器
open -a Simulator

# 2. 截图
# 快捷键：Cmd + Shift + 4，然后选择模拟器窗口

# 3. 调整尺寸
# 使用 Preview 或其他工具调整为 1242x2208px
```

---

### 方法 2：使用真机截图

#### Android 真机

```bash
# 1. 连接手机
adb devices

# 2. 截图
adb shell screencap -p /sdcard/screenshot.png
adb pull /sdcard/screenshot.png ~/Desktop/

# 3. 调整尺寸为 1080x1920px
```

#### iOS 真机

```
# 1. 连接 iPhone
# 2. 截图：电源键 + 音量加键
# 3. 从照片导出
# 4. 调整尺寸为 1242x2208px
```

## 四、截图优化技巧

### 1. 内容准备

**测试数据准备**：
- 确保测试数据完整（用户信息、帖子内容等）
- 使用真实的中文内容
- 避免使用"测试"、"test"等占位符

**测试账号准备**：
- 使用"测试创业者01"等测试账号
- 确保账户有完整的数据（帖子、关注、收藏等）

---

### 2. 截图时机

**最佳时机**：
- 内容加载完成后
- 动画播放完成后
- 键盘弹出或收起后
- 页面切换动画完成后

**避免时机**：
- 内容加载中
- 动画播放中
- 错误提示显示时
- 弹窗显示时

---

### 3. 尺寸调整

**尺寸要求**：
- Android：1080x1920px（或更高，保持 9:16 比例）
- iOS：1242x2208px（iPhone 6.7"）

**调整方法**：
```bash
# 使用 ImageMagick
convert screenshot.png -resize 1080x1920! output.png

# 使用 ffmpeg
ffmpeg -i screenshot.png -vf scale=1080:1920 output.png

# 使用在线工具
# https://www.iloveimg.com/resize-image
```

---

### 4. 图片优化

**压缩优化**：
```bash
# 使用 ImageMagick 压缩
convert output.png -quality 90 output_compressed.png

# 使用 TinyPNG
# https://tinypng.com/
```

**格式转换**：
- PNG → JPG（可以减少文件大小）
- JPG → PNG（保持透明背景）

---

### 5. 添加标注（可选）

使用工具：
- Figma：添加文字标注、箭头等

标注内容：

## 八、常见问题

### Q1: 截图尺寸不对怎么办？

A: 使用图片编辑工具调整尺寸，保持 9:16 的宽高比。

### Q2: 截图内容不完整怎么办？

A: 确保模拟器/真机的分辨率足够高，重新截图。

### Q3: 如何去除系统状态栏？

A:
- Android：使用全屏模式，或在编辑时裁剪
- iOS：使用模拟器截图时选择无状态栏模式

### Q4: 截图文件太大怎么办？

A: 压缩图片或转换为 JPG 格式。

### Q5: 需要为不同语言准备截图吗？

A: 如果应用支持多语言，需要为每种语言准备截图。遇合目前仅支持中文，使用中文截图即可。
